# 先手投降功能所需图片说明

本目录存放国服先手投降功能所需的所有图片。

## 图片列表

### 1. 主页面相关
- **duel_button.png** - 主页面的"决斗"按钮（重要：用于判断是否回到主页面）
- **home_mark.png** - 主页面标志（可选，作为备用判断）

### 2. 回合检测相关
- **round.png** - "回合"文字图片（用于检测是否已进入决斗）
- **round1.png** - "回合1"的图片
- **round2.png** - "回合2"的图片

### 3. 回合归属相关
- **your_turn.png** - "你"的图片（表示当前是自己的回合）
- **opponent_turn.png** - "对手"的图片（表示当前是对手的回合）

### 4. 投降操作相关
- **menu.png** - 决斗中的菜单按钮
- **surrender.png** - 菜单中的"认输"按钮
- **confirm.png** - 确认投降的"确定"按钮
- **next_step.png** - "下一步"按钮

### 5. 对手投降相关
- **opponent_surrender.png** - "对手认输"的提示图片

## 截图建议

### 截图时机：
1. **决斗按钮**：在主页面时截取，这是判断是否回到主页面的关键图片
2. **回合相关图片**：进入决斗后，在回合开始时截取
3. **投降相关图片**：在决斗中打开菜单，进入投降确认界面时截取
4. **确定和下一步**：在决斗结束后的奖励界面截取

### 截图要求：
- 图片应该清晰，包含完整的目标元素
- 避免截取会变化的部分（如动画中的图片）
- 建议使用较高的图片质量，以提高识别准确度
- **确定**和**下一步**按钮可能在多个界面出现，建议截取最常见的样式

## 重要说明

### 关于返回主页面的逻辑：
脚本会在投降后不断点击"确定"或"下一步"按钮，直到检测到"决斗按钮"（duel_button.png）出现，才认为已经返回主页面。因此：

1. **duel_button.png** 是最重要的图片，必须准确截取
2. **confirm.png** 和 **next_step.png** 也很重要，用于跳过奖励和对话界面
3. 如果返回主页面过程中出现其他按钮，可以补充截图

## 使用方法

将截取好的图片按照上述文件名保存到本目录即可。

## 注意事项

- 所有图片格式为 PNG
- 图片识别默认置信度为 0.9，如果识别不准确可以在代码中调整
- 确保截图时游戏分辨率与运行时一致
- 建议先运行一次，观察日志输出，看哪些图片识别不准确，然后重新截取
